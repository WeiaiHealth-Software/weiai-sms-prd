# 系统 UI 规范总纲

本文档用于统一当前后台系统页面的结构、标题归属、表格工具栏、表单页、按钮层级、标签状态与间距规范。后续新增页面、重构页面、补齐原型时，若无特殊业务说明，统一以本文为默认标准。

## 1. 适用范围

- 适用于当前 `src/` 下所有后台管理端页面。
- 当前争议处理顺序：`DESIGN.md` > `src/pages/system/UiSpec.tsx` 规范演示页 > 已落地页面示例。
- 若业务确需偏离，必须说明原因，且优先在同一模块内保持一致。

## 2. 页面结构

### 2.1 总体骨架

系统统一采用「左侧导航 + 右侧内容区」后台布局：

- 左侧：Sidebar 导航。
- 右侧顶部：Header。
- 右侧内容区：页面主体内容，统一放在主滚动区域内。

页面结构默认分为 3 层：

1. Header 层：承载当前模块标题、子级标题、必要说明。
2. Page Body 层：承载页面真正内容，不重复出现 Header 同级大标题。
3. Card / Section 层：承载筛选、表格、表单、说明、详情块等具体内容。

### 2.2 页面容器规范

- 页面主内容区默认使用纵向编排，推荐 `gap-6`。
- 一级内容卡片统一使用白底卡片，推荐：
  - 圆角：`rounded-2xl`
  - 边框：`border border-gray-100` 或 `border border-slate-200`
  - 背景：`bg-white`
- 页面不要直接把大量内容裸露在灰底背景上，优先通过卡片组织信息层级。

## 3. Header 标题归属

### 3.1 标题归属原则

- 页面标题属于 Header，不属于页面内容卡片。
- 页面进入某个模块后，Header 负责表达「当前系统 / 当前页面 / 当前子页面」。
- 页面主体内部不再重复放一个与 Header 同级的大标题。

### 3.2 Header 表达方式

- 一级标题：当前模块名，例如 CRM、门店管理、系统规范。
- 二级标题：当前具体页面，例如客户档案、客户档案详情、发起回访。
- 可选描述：只在详情页、新建页、复杂流程页补充一句解释。

### 3.3 禁止事项

- 禁止在表格卡片顶部再次写一个与 Header 一样的页面大标题。
- 禁止 Header 写模块名，页面正文再写一遍同级页面名，造成“双标题”。
- 禁止一个页面同时存在 2 个视觉权重接近的主标题。

## 4. 表格页规范

### 4.1 标准组成

标准表格页从上到下按以下顺序组织：

1. Header 标题
2. 工具栏卡片
3. 表格卡片 / 表格主体
4. 分页区

### 4.2 工具栏布局

表格页工具栏必须遵循以下结构：

- 左侧放筛选、搜索、分段筛选、日期范围等查询控件。
- 右侧放 Action，如新增、导出、批量处理。
- 右侧 Action 区左边必须有一条竖线分隔。

标准理解为：

- 左边是“找数据”。
- 右边是“做动作”。

这样可以让用户快速识别查询区与操作区，避免工具栏所有按钮混成一排。

### 4.3 工具栏视觉要求

- 工具栏建议单独作为一张卡片，不与表格表头混写。
- 查询区与 Action 区之间使用竖向分隔线：
  - 推荐尺寸：`h-6 w-px`
  - 推荐颜色：`bg-gray-200` / `border-slate-200`
- 筛选控件与按钮间距统一，推荐 `gap-3` 或 `gap-4`。
- 主操作按钮放在 Action 区最右侧。

### 4.4 表格内容规范

- 表头统一弱化，使用浅灰底、较小字号、较弱文字色。
- 表格行 hover 可轻微高亮，提升可扫读性。
- 行内主操作优先使用浅色按钮，如“查看详情”“编辑”。
- 行内危险操作使用红色文字按钮，并配合二次确认弹窗。
- 空状态必须保留单独一行或单独卡片提示，不能直接留白。

## 5. 表单页规范

### 5.1 表单页结构

表单页统一采用「顶部动作 + 多 Section 表单块」结构：

1. 顶部返回 / 保存 / 提交 / 完成等动作
2. 表单主体 Section
3. 底部重复操作区（长表单建议保留）

### 5.2 表单块组织

- 一个业务语义对应一个 Section，例如“客户信息”“回访信息”“收费详情”。
- 每个 Section 使用白底卡片承载，不建议把所有字段堆成一整页长表单。
- 字段说明文字弱化，字段值与输入控件保持稳定对齐。
- 长表单建议按 2 列或 3 列栅格排布；说明型文本域独占整行。

### 5.3 控件规范

- Input / Select / Date / Textarea 统一使用圆角与浅边框。
- Focus 使用主色边框与轻微描边高亮。
- 同类字段的高度应一致，避免一个页面出现多套输入框高度。
- 文本域仅用于长内容；短字段不要滥用多行文本框。

### 5.4 顶部与底部动作

- 表单页顶部动作推荐：返回 + 次按钮 + 主按钮。
- 长表单底部可重复一组关键按钮，避免滚动到顶部再提交。
- 主按钮文案必须表达结果，例如“提交回访”“完成并返回”“保存草稿”。

## 6. 按钮层级

统一使用 4 级按钮语义：

### 6.1 一级按钮

- 用途：页面主操作、流程推进、最终提交。
- 样式：品牌主色实底。
- 示例：新增、保存、提交、完成并返回。

### 6.2 二级按钮

- 用途：辅助操作、返回、取消、导出、查看次流程。
- 样式：白底、灰边、深灰文字。

### 6.3 三级按钮

- 用途：表格行内轻操作、弱强调操作。
- 样式：浅色底按钮，如浅蓝底的“查看详情”“编辑”。

### 6.4 危险按钮 / 危险文字

- 用途：删除、终止、不可逆操作。
- 行内优先红色文字按钮。
- 弹窗确认按钮使用红色实底按钮。

### 6.5 使用原则

- 一个区域内通常只允许 1 个一级按钮。
- 一级按钮必须出现在操作流的最终落点。
- “取消 / 返回”永远不要与主按钮抢视觉层级。
- 带图标按钮统一图标在左、文案在右，推荐 `gap-2`。

## 7. 标签状态规范

### 7.1 标签用途

标签只承担两类信息：

- 状态：如进行中、已结束、待复查、草稿、已录入。
- 分类：如会员类型、客户来源、项目分类。

### 7.2 标签样式

- 统一为浅底 + 深字 + 细边框。
- 推荐形态：
  - 常规分类标签：`rounded-full`
  - 紧凑状态标签：`rounded-md` 或 `rounded-full`
- 不使用高饱和纯色实底标签作为默认方案。

### 7.3 颜色语义

- 蓝色：默认主状态、普通高亮、系统主推荐项。
- 绿色：成功、正常、进行中、已录入。
- 橙色 / 黄色：提醒、待处理、临时关注。
- 灰色：中性、已结束、未启用、缺省状态。
- 红色：风险、删除、终止、异常。
- 紫色：会员层级、特殊分类等非风险增强信息。

### 7.4 状态命名要求

- 状态名优先使用业务真实语义，不使用模糊词。
- 同一业务状态跨页面必须同名同色，不允许“待跟进 / 跟进中 / 处理中”混用表达同一个状态。

## 8. 间距与分隔

### 8.1 页面层级间距

- 页面级模块之间：推荐 `24px`，对应 `gap-6`。
- 卡片内部普通区块之间：推荐 `16px` 到 `20px`。
- 表单字段与标题说明之间：推荐 `8px`。
- 行内按钮 / 标签 / 小控件间距：推荐 `8px` 到 `12px`。

### 8.2 分隔方式

- 大区块分隔优先使用上下留白，不滥用粗重横线。
- 卡片内子区块需要边界时，使用浅色边框或浅底色。
- 表格工具栏的查询区与 Action 区，必须使用竖线分隔。
- Header 右侧系统动作、用户信息之间可使用短竖线辅助分隔。

### 8.3 一致性要求

- 同一页面不要同时出现多种完全不同的圆角体系。
- 同一页面不要混用过多间距尺度，优先围绕 8 / 12 / 16 / 20 / 24 组织。
- 分隔线只在有信息分区价值时使用，不为“装饰”而加线。

## 10. 常见错误

以下问题在后续设计与开发中默认视为常见错误，应主动避免：

### 10.1 标题错误

- Header 已经有标题，页面正文卡片里再写一个同级主标题。
- 一个页面出现多个视觉相近的大标题，无法判断主次。
- 模块名、页面名、卡片名三层全部写成同样字号和字重。

### 10.2 表格页错误

- 把筛选、导出、新增全部塞在一行里，没有左筛选右 Action 分区。
- Action 左侧没有竖线，导致工具栏层级不清。
- 工具栏和表头混在一起，页面结构发散。
- 表格行主操作与危险操作视觉层级颠倒。

### 10.3 表单页错误

- 所有字段连续堆叠，没有 Section 分组。
- 页面顶部只有保存按钮，没有返回或取消，用户缺少退路。
- 字段高度、边框、圆角不统一，出现多套输入框体系。
- 长表单只有顶部操作，滚动到底部无法快速提交。

### 10.4 按钮错误

- 一个区域出现多个主色实底按钮，不知道哪个是主操作。
- 把删除做成与保存同级的主色按钮。
- 返回、取消使用过强视觉权重。

### 10.5 标签错误

- 同一类状态跨页面颜色不一致。
- 用彩色实底标签充当默认展示，视觉过重。
- 分类标签和状态标签样式完全混乱，没有统一规则。

### 10.6 间距错误

- 页面区块过于拥挤，信息堆在一起。
- 同一页混用 10 多种间距与圆角值。
- 明明应该靠留白分隔，却滥用边框和分割线。

## 11. 默认执行要求

- 新页面评审时，先检查是否符合 Header 归属规则。
- 新表格页评审时，先检查是否满足“左筛选、右 Action、Action 左竖线”。
- 新表单页评审时，先检查是否有清晰 Section 与明确按钮层级。
- 新标签和状态引入时，先复用既有语义和颜色，不新增无必要状态色。
- 如遇规范冲突，优先回到本文件与 `src/pages/system/UiSpec.tsx` 对齐。
