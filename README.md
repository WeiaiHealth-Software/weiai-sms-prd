# 惟翎门店管理系统 - 项目文档

## 项目概述

惟翎门店管理系统是一个专业的视光诊疗管理平台，包含门店管理模块和小程序预约系统。该系统采用HTML、CSS、JavaScript和TailwindCSS原生代码开发，主题色为蓝色（#3b82f6），旨在为医疗机构提供高效的预约、接诊和患者管理解决方案。

## 核心功能模块

### 1. 前台面板（前台工作人员）

**主要功能**：

- **员工登录**：选择员工 → 输入4位密码 → 进入工作面板
- **预约管理**：实时显示今日预约列表，支持预约确认
- **快速代预约**：为现场患者快速创建预约
- **手动加号**：紧急情况下为医生加号
- **号源管理**：查看各医生各时间段的号源情况
- **统计面板**：显示今日预约总数、已确认、待确认、已完成数量

**登录凭证示例**：

| 员工名称 | 密码 |
| -------- | ---- |
| 张三     | 1234 |
| 李四     | 5678 |
| 王五     | 9012 |

**关键特性**：

- 实时弹窗提醒患者线上预约
- 防超约机制，显示真实可约号数
- 支持按类型（初诊/复查）智能分配名额
- 管理员可突破限制手动加号

---

### 2. 医生面板（医生/视光师）

**主要功能**：

- **员工登录**：选择医生 → 输入4位密码 → 进入工作面板
- **接诊队列**：自动显示今日接诊队列，按预约顺序排序
- **患者详情**：一屏展示患者信息（姓名、预约类型、既往病史、是否带镜等）
- **一键开单**：支持生成处方、检查单、验光单
- **快速模板**：预设常用模板（近视矫正、散光矫正、老花镜等）
- **统计面板**：显示待接诊、接诊中、已完成患者数量

**登录凭证示例**：

| 医生名称 | 密码 |
| -------- | ---- |
| 医生A    | 1234 |
| 医生B    | 5678 |
| 医生C    | 9012 |

**关键特性**：

- 患者信息一屏展示，避免信息遗漏
- 一键开单支持多种单据类型
- 快速模板加速工作流程
- 不涉及占位和流程流转，专注问诊和下医嘱

---

### 3. 视光师面板（视光师）

**工作模式**：

#### 3.1 移动模式

- **特点**：Pad跟随视光师移动，需要手动选择诊室
- **流程**：
  1. 登录后选择工作模式（移动模式）
  2. 选择诊室（201、202、301等）
  3. 点击"开始接诊"开始服务
  4. 诊室屏幕自动显示"视光师A | 患者名 | 验光中"
  5. 点击"结束接诊"释放诊室，可切换到下一个诊室
- **优势**：灵活，支持跨诊室工作
- **劣势**：需要多一步选择诊室的操作

#### 3.2 固定模式

- **特点**：Pad固定在诊室，视光师进来直接登录即占用
- **流程**：
  1. 登录后选择工作模式（固定模式）
  2. 输入诊室号（如201）
  3. 点击"开始接诊"开始服务
  4. 诊室屏幕自动显示"视光师A | 患者名 | 验光中"
  5. 点击"结束接诊"释放诊室
- **优势**：简单快速，无需选择诊室
- **劣势**：不灵活，固定在一个诊室

**登录凭证示例**：

| 视光师名称 | 密码 |
| ---------- | ---- |
| 视光师A    | 1234 |
| 视光师B    | 5678 |
| 视光师C    | 9012 |

**关键特性**：

- 智能诊室冲突提示：如果诊室被占用，系统会提示是否切换
- 实时状态显示：显示当前接诊患者和诊室号
- 两种模式灵活切换，适应不同场景

### 4. 小程序（患者端）

**主要功能**：

#### 4.1 首页

- **快速预约入口**：一键进入预约流程
- **功能卡片**：我的预约、家庭成员、帮助中心
- **最近预约**：显示最近的预约记录

#### 4.2 预约流程

1. **选择医院**：支持多家门店选择
2. **选择医生**：网格展示医生列表
3. **选择日期**：日期选择器，支持未来两周
4. **选择时间段**：显示每个时间段的剩余号数，已满显示"已满"
5. **选择就诊类型**：初诊 / 复查
6. **选择号源类型**：普通号（¥41）/ 特需号（¥300）
7. **确认预约**：提交预约，等待前台确认

#### 4.3 个人中心

- **用户信息**：显示用户头像、姓名、电话
- **我的预约**：查看所有预约记录，支持按状态筛选
- **我的家庭组**：管理家庭成员，支持添加/删除
- **我的就诊人**：管理就诊人信息
- **帮助中心**：常见问题和使用指南
- **意见反馈**：用户反馈通道
- **关于我们**：应用信息和版本号

#### 4.4 患者个人码

- **圆形按钮**：底部导航栏居中显示
- **二维码展示**：患者个人码，便于视光师快速操作
- **保存功能**：支持保存到相册

**关键特性**：

- 微信一键登录
- 家庭账户支持，可添加多个家庭成员
- 多设备支持，扫码绑定新设备
- 实时显示号源剩余数量
- 预约成功自动接收通知

## 数据流向

### 预约流程数据流

```
患者小程序预约
    ↓
前台面板实时弹窗提醒
    ↓
前台确认预约
    ↓
患者自动接收通知
    ↓
医生面板显示接诊队列
    ↓
视光师面板显示患者信息
    ↓
完成接诊
```

### 号源管理数据流

```
系统初始化号源（每半小时一个时间段，最多3人）
    ↓
患者预约时实时锁定号源
    ↓
防止超约，显示真实可约号数
    ↓
管理员可手动加号（突破限制）
    ↓
患者端永远只看到"已满"（不显示加号）
```

## 关键业务规则

### 1. 号源管理规则

- **时间段**：每半小时一个时间段（08:00-08:30、08:30-09:00等）
- **容纳人数**：每个时间段最多3人
- **就诊类型**：初诊和复查有不同的时间占用规则
- **号源类型**：普通号（¥41）和特需号（¥300）
- **防超约**：系统实时计算可用号源，防止人工代约导致超额
- **手动加号**：管理员可在满员情况下加号，但患者端永远只看到"已满"

### 2. 预约流程规则

- **自助预约**：患者通过小程序微信一键登录后预约
- **代预约**：前台可为现场患者快速代预约
- **确认机制**：患者预约后前台收到弹窗提醒，确认后患者自动接收通知
- **建档要求**：前台代预约时需先为患者完成注册建档

### 3. 家庭账户规则

- **账号绑定**：账号绑定家庭ID，支持添加多个家庭成员
- **多设备支持**：同一账号可在多台设备登录，通过扫码绑定新设备
- **解决问题**：解决家长轮流带孩子就诊时手机号不一致的问题

### 4. 员工登录规则

- **统一登录逻辑**：所有面板（前台/医生/视光师）统一登录方式
- **身份识别**：选择角色 → 输入4位密码
- **权限管理**：后端排班系统知道"今天谁当班"，自动开放对应权限
- **状态管理**：员工登陆成功后，系统中员工管理面板的状态变为"在线"或"服务中"

## 安全性考虑

### 1. 身份验证

- 4位数字密码验证
- 后端排班系统权限控制
- 会话超时自动退出

### 2. 数据保护

- 输入验证和清理
- 防止XSS攻击
- HTTPS传输（生产环境）

### 3. 访问控制

- 基于角色的访问控制（RBAC）
- 前台只能看到预约管理
- 医生只能看到自己的患者
- 视光师只能看到分配的患者

## 常见问题

### Q1: 如何重置员工密码？

A: 密码由排班系统预配，每天自动更新。如需重置，请联系系统管理员。

### Q2: 患者预约后多久会收到通知？

A: 前台确认预约后，患者会立即自动接收通知。

### Q3: 如何处理号源超约情况？

A: 系统会实时锁定可用号源，防止超约。如需加号，管理员可手动加号。

### Q4: 视光师可以同时在多个诊室工作吗？

A: 不可以。视光师在移动模式下可以切换诊室，但同一时间只能在一个诊室工作。

### Q5: 小程序支持多少个家庭成员？

A: 理论上无限制，但建议不超过10个以保证用户体验。

### Q6: 如何导出预约数据？

A: 当前版本不支持导出，建议在后续版本中添加此功能。
