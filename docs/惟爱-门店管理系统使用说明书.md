## 1. 系统概览

### 1.1 系统形态

- **后台管理系统（PC 端）：** 面向门店经营与运营人员，用于人员、诊室、排班、客户档案等管理。
- **小程序端（患者端）：** 面向患者/家长，支持登录、查看服务入口、扫码注册并新增客户档案（演示）。

### 1.2 账号与权限（当前版本）

- 当前版本为演示态：默认已登录到门店后台（单门店模式），页面右上角显示当前登录人员与门店信息。
- 小程序端采用演示登录：点击「微信用户一键登录」即可进入首页。

## 3. 门店管理模块

门店管理模块用于维护门店的基础运营数据，主要包括：人员管理、诊室管理、门店排班。

### 3.1 人员管理

**入口：** 左侧导航 → 门店管理 → 人员管理

**主要功能：**

- **搜索与筛选：** 支持按姓名/职称搜索，按角色、在职状态筛选。
- **录入人员：** 新增医生/视光师/视训师/前台等岗位人员信息。
- **编辑信息：** 修改人员基础信息、擅长描述、入职时间与状态等。

**典型使用场景：**

- 新增一名医生/视光师后，将其加入排班，开放预约号源。
- 人员离职时，将状态调整为「已离职」，避免继续排班与指派。

> 建议截图：人员管理列表（含搜索、角色筛选、在职筛选、人员卡片/表格、右侧「编辑」按钮）。  
> 图片路径（占位）：`./assets/weiai-manual/store-personnel.png`

### 3.2 诊室管理

**入口：** 左侧导航 → 门店管理 → 诊室管理

**主要功能：**

- **分类查看：** 支持按「全部 / 诊室 / 验光室 / 训练室」切换查看。
- **新增诊室：** 维护门牌号、诊室名称、类型、容量、状态、功能描述等信息。
- **诊室状态：** 用于反映诊室是否可用（启用中/维修中/已废弃）。

**典型使用场景：**

- 新增「角塑验配」验光室，设置类型为「验光室」，并配置功能描述。
- 诊室检修期间将状态改为「维修中」，排班时避免分配。

### 3.3 门店排班

**入口：** 左侧导航 → 门店管理 → 门店排班

**主要功能：**

- **排班日历：** 按日期查看当天排班。
- **班次与诊室分配：** 展示上午/下午班人员与诊室分配。
- **号源时间轴：** 展示医生/视光师在时间轴上的可预约号源（演示）。

**典型使用场景：**

- 每周设置下一周的医生/视光师出诊与验光安排。
- 高峰时段增加号源，提升接诊能力。

## 4. 客户档案（CRM）模块（客户管理）

客户档案模块用于记录门店客户（患者）信息。客户来源包括：小程序扫码注册（新增档案）。

### 4.1 档案管理（列表）

**入口：** 左侧导航 → 客户档案（CRM）→ 档案管理

**列表字段说明（核心）：**

- **客户信息：** 姓名、年龄、性别、手机号、患者编号等。
- **就诊日期/类型/数据记录/诊断：**
  - 若客户仅完成小程序注册、尚未到店建档或产生就诊记录，则该类字段显示为 `--`。
  - 若客户已有就诊记录，则显示最近一次就诊相关信息。

**常用操作：**

- **搜索：** 支持按姓名/手机号/患者编号搜索。
- **筛选：** 支持按就诊类型、日期范围筛选（演示）。
- **新增档案：** 点击右上角「新增档案」进入建档流程（后台侧建档）。
- **查看档案：** 进入客户档案详情页。

> 🖼️ 配图占位：档案管理列表页  
> 建议截图：顶部搜索筛选条 + 列表前几条（包含 `--` 的“新客户”行）+ 右上角「新增档案」。

### 4.2 查看档案（未到店客户）

**入口：** 档案管理列表 → 点击某客户右侧「查看档案」

**说明：**

- 对于「仅注册、未到店」的客户，系统只展示其基础信息。
- 就诊记录、视光训练记录、配镜记录、预约记录、回访记录、消费记录等均为空，并提示「暂无档案信息」。

### 4.3 客户信息字段（扫码注册）

扫码注册时填写信息分为：**基本信息**（必填为主）与 **病史信息**（选填）。

**基本信息：**

- 姓名（必填）
- 年龄（必填）
- 性别（必填）
- 联系方式：手机号（必填）
- 身份证号（选填）

**病史信息（选填）：**

- 眼部疾病史：下拉选择，默认「无」
- 眼部手术史：无 / 左眼 / 右眼 / 双眼
  - 若选择非「无」，需补充「手术史描述」（必填）

### 5.3 新增档案（扫码注册）

**入口：** 小程序首页 → 眼健康守护计划 → 「新增档案」

**门店识别规则：**

- 页面顶部显示「门店（扫码自动识别）」。
- 门店值从二维码参数/URL query 读取（例如 `storeId=默认门店`），不可手动编辑。

**填写与校验规则：**

- 基本信息必填：姓名、年龄、性别、手机号
- 身份证号选填
- 病史信息选填
- 若眼部手术史选择非「无」，则手术史描述必填

**提交结果：**

- 提交成功提示「建档成功」
- 返回小程序首页
- 后台档案管理列表可看到新客户（演示数据存储在浏览器本地）

### 5.4 小程序与后台的数据联动（演示说明）

当前演示版本为了便于快速联调，使用浏览器本地存储（localStorage）模拟“扫码注册 → 后台可见”的联动效果：

- 小程序端提交后写入本地客户数据
- 后台「档案管理」列表读取并展示该数据

> 生产环境中通常会替换为真实接口：小程序提交到服务端，后台从服务端拉取同一份客户数据。

## 6. 常见问题（FAQ）

### 6.1 为什么有些客户在列表里显示 `--`？

这类客户通常是「小程序扫码注册后尚未到店」：系统只有基础信息，没有就诊日期、就诊类型、检查数据与诊断内容，因此显示为 `--`。

### 6.2 小程序新增档案后，后台列表没看到？

演示版本的数据写入在浏览器 localStorage 中：

- 尝试刷新后台页面；
- 确认是否在同一浏览器环境（同一域名、同一设备）下操作；
- 若你清理了浏览器缓存或 localStorage，演示数据也会随之清除。

### 6.3 如何清空演示数据？

打开浏览器开发者工具（DevTools）并清理 localStorage 中相关 key（示例）：

- `weiai.crm.patients`：客户档案演示数据
- `weiai.miniprogram.isLoggedIn`：小程序演示登录态
